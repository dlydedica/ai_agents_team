## Условные обозначения

| Уровень | Описание | Ожидаемый результат |
|---------|----------|---------------------|
| ✅ | Грейд доступен для этой роли | Может работать на этом уровне |
| — | Грейд не предусмотрен | Роль не требует этого уровня |
| Пусто | Нет сотрудника этого грейда | Можно нанять при необходимости |

## Как используются грейды

1. **При постановке задачи** CEO указывает нужный грейд
2. **При performance review** HR оценивает, соответствует ли сотрудник грейду
3. **При развитии** HR планирует повышение грейдов (Junior → Middle → Senior)
4. **При создании subagent'а** учитывается грейд для ограничения инструментов
