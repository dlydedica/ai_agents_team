## 1. Видение (Vision)

> **AI DevCorp — полностью автономная корпорация AI-агентов, способная выполнить любую задачу разработки от идеи до продакшена без участия человека.**

### Цели на 12 месяцев

| Срок | Цель | Критерий |
|------|------|----------|
| 3 мес | **MVP** — команда выполняет простые задачи end-to-end | 1 задача < 10 шагов |
| 6 мес | **Продакшен** — команда выполняет сложные задачи с качеством >90% | Code review pass rate |
| 12 мес | **Автономия** — команда получает задачу и садится писать код без уточнений | Zero clarification loops |

## 3. Стратегические направления (Pillars)

### Pillar A: 🧠 Автоматическая оркестрация
*Цель: CEO сам вызывает subagents по цепочке*

**Ключевые инициативы:**
- AI: Научить CEO читать `task_store` и вызывать subagents последовательно
- AI: Добавить самодиагностику — CEO проверяет свою работу после каждого шага
- Dev: Реализовать `team.py orchestrate` — реальный запуск цепочки

**Метрика:** % задач, выполненных без ручного вмешательства

### Pillar B: 🤖 Качество и тестирование
*Цель: Каждый коммит проверяется CI/CD*

**Ключевые инициативы:**
- DevOps: GitHub Actions — автотесты + линтер на каждый PR
- QA: Расширить тесты — pytest на server.py, dashboard
- QA:Добавить mutation testing

**Метрика:** Coverage >85%, 0 critical bugs в релизе

### Pillar C: 📊 Data-Driven развитие
*Цель: Решения о развитии на основе метрик*

**Ключевые инициативы:**
- Data: Сбор метрик производительности отделов
- Data: Дашборд с трендами (скорость, качество, загрузка)
- HR: Performance review по завершению каждой задачи
- HR: Skill gap analysis

**Метрика:** Время выполнения задачи, качество кода, загрузка отделов

### Pillar D: 🔗 Интеграция и экосистема
*Цель: Любой проект подключает AI DevCorp в 1 команду*

**Ключевые инициативы:**
- Dev: CLI-команда для быстрой установки (pip install или curl)
- Docs: API-документация для MCP-сервера
- Marketing: Примеры использования, шаблоны задач

**Метрика:** Количество интеграций, время установки < 1 мин

### Pillar E: 🧠 Память и контекст
*Цель: Команда помнит прошлые задачи и учится на ошибках*

**Ключевые инициативы:**
- Data: Persistent memory — хранение результатов задач
- R&D: Semantic search по прошлым задачам
- R&D: Автоматические ретроспективы

**Метрика:** Время повторной задачи < 50% от первой

## 8. Модель развития: Global vs Local

Команда развивается на двух уровнях одновременно.

### 🌍 Global (Core — этот репозиторий)

Изменения, которые применяются **ко всем проектам** сразу.

| Что развивается | Где | Как |
|----------------|-----|-----|
| 🧠 Агенты отделов | `.github/agents/` | Улучшение промптов, новые роли |
| 🤖 MCP-сервер | `mcp-server/` | Новые инструменты, фиксы |
| 🔄 Handoff-протоколы | `interactions/` | Оптимизация цепочек |
| 📊 Дашборд | `dashboard/` | Новые метрики |
| 🧪 Тесты | `mcp-server/tests/` | Расширение покрытия |
| 🏢 Структура отделов | `departments/` | Новые отделы, роли |

**Механизм:** `git push` в `ai_agents_team` → `git submodule update` во всех проектах

### 📍 Local (в проекте заказчика)

Данные, уникальные для конкретного проекта.

| Что развивается | Где | Как |
|----------------|-----|-----|
| 📝 История задач | `memory/data/` | TF-IDF, семантика |
| 👥 HR-отчёты | `docs/hr/` | Performance Review |
| 📊 Метрики проекта | `memory/data/` | Дашборд + тренды |
| 📋 Шаблоны | `templates/` | Специфичные для проекта |

**Механизм:** Эти данные **не пушатся** в core (в `.gitignore`).

### 🔄 Цикл: Local → Global

```
Проект → найден баг в MCP-сервере
       → фикс коммитится в submodule
       → git push в ai_agents_team (main)
       → все проекты получают обновление

Проект → накоплено 50 задач
       → HR видит: Development перегружен
       → HR рекомендует: добавить разработчика
       → изменение global: новая роль в competence_matrix.md

Проект → новая технология (например, Kafka)
       → R&D изучает, добавляет в tech radar
       → обновление global: новый скил в competence_matrix.md
```
