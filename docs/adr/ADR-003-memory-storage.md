# ADR-003: Хранение памяти

**Статус:** Принято
**Дата:** 2026-06-28
**Автор:** 🏗️ Architecture

## Контекст
Команда должна помнить прошлые задачи, чтобы учиться на ошибках и находить похожие решения.

## Решение
Использовать **TF-IDF + Cosine Similarity** (pure Python) для семантического поиска.

Хранилище:
- `memory/data/knowledge.json` — задачи, решения, уроки
- `memory/data/idf.json` — IDF-индекс
- `memory/data/vocabulary.json` — словарь токенов

## Альтернативы
- **Embeddings API (OpenAI)** — требуется API-ключ, платно
- **SQLite FTS5** — только полнотекстовый, не семантический
- **FAISS / векторные БД** — избыточно для текущего объёма

## Последствия
- Zero external dependencies — работает из коробки
- TF-IDF достаточно точен для поиска по задачам (cosine >0.5)
- При росте >1000 задач можно перейти на embeddings
