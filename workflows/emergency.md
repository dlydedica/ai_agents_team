# Emergency Workflow (Срочный процесс)

Используется для горящих задач: критических багов, срочных хотфиксов.

## Отличия от Default

| Параметр | Default | Emergency |
|----------|---------|-----------|
| Architect | Полный цикл | Пропускается (упрощённый план) |
| Developer | Полная реализация | Минимальное изменение |
| Reviewer | Полное ревью | Quick check |
| Documenter | Полная документация | Только changelog |
| Тесты | Полный набор | Только smoke-тесты |

## Flow

```
Orchestrator (быстрый анализ)
       │
       ▼
Developer (минимальное изменение)
       │
       ▼
Reviewer (quick check)
       │
       ▼
Orchestrator (результат)
```

## Шаги

### Шаг 1: Быстрый анализ (Orchestrator)
- Определить критичность
- Оценить impact
- Определить минимальное изменение

### Шаг 2: Хотфикс (Developer)
- Минимальное изменение для исправления
- Без рефакторинга
- Без новой функциональности

### Шаг 3: Quick Review (Reviewer)
- Проверка только изменённых строк
- Убедиться, что фикс не ломает существующее

### Шаг 4: Результат (Orchestrator)
- Собрать результат
- Отметить в changelog
