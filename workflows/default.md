# Default Workflow (Стандартный процесс)

Используется для большинства задач — от новых проектов до фич и багфиксов.

## Flow

```
🧠 CEO
   │
   ├──► 🏭 Product ──────► спецификация
   ├──► 🏗️ Architecture ──► проект
   ├──► 💻 Development ───► код
   ├──► 🧪 QA ───────────► тесты
   ├──► ⚙️ DevOps ───────► инфраструктура
   ├──► 🎨 Design ───────► макеты
   ├──► 📖 Docs ─────────► документация
   ├──► 🛡️ Security ─────► аудит безопасности
   ├──► 📊 Data ─────────► данные и аналитика
   ├──► 🔬 R&D ──────────► исследования (при необходимости)
   ├──► ⚖️ Legal ────────► compliance (при необходимости)
   ├──► 📣 Marketing ────► стратегия продвижения (при необходимости)
   └──► 👥 HR ───────────► анализ и рекомендации
         │
         ▼
   🧠 CEO (сборка отчёта)
```

## Шаги

### Шаг 1: Анализ задачи (CEO)
- Определить тип задачи (новый проект / фича / багфикс / рефакторинг)
- Определить, какие отделы нужны
- Поставить задачи Heads

### Шаг 2: Каждый Head управляет своим отделом

#### 🏭 Product
- PM анализирует требования → Analyst готовит спецификацию
- Результат: `specification.md`

#### 🏗️ Architecture
- System Architect проектирует → Solution Architect детализирует
- Результат: `architecture.md`, ADR, API-контракты

#### 💻 Development
- Tech Lead распределяет → Developer-ы реализуют
- Результат: код, unit-тесты

#### 🧪 QA
- QA Lead определяет стратегию → Testers проверяют
- **Цикл:** баги → возврат в Development

#### ⚙️ DevOps
- DevOps Lead настраивает инфраструктуру → SRE деплоит
- Результат: CI/CD, окружение

#### 🎨 Design
- Design Lead утверждает концепцию → UI/UX Designer рисует
- Результат: макеты, прототипы

#### 📖 Docs
- Technical Writer пишет документацию
- Результат: README, API docs

#### 👥 HR
- HR Lead анализирует выполнение задачи
- Оценивает загрузку отделов и качество работы
- Формирует рекомендации по развитию команды
- Результат: performance review, рекомендации

#### 🛡️ Security
- Security Lead проводит аудит безопасности кода и инфраструктуры
- Проверяет уязвимости (SAST/DAST)
- Оценивает compliance (GDPR, 152-ФЗ)
- Результат: security report

#### 📊 Data
- Data Lead оценивает потребности в данных и аналитике
- Data Engineer настраивает пайплайны (если нужно)
- BI Analyst готовит дашборды метрик
- Результат: data pipeline, дашборды

#### 🔬 R&D
- Привлекается для оценки новых технологий
- Проводит PoC и бенчмаркинг
- Готовит Technology Radar
- Результат: recommendation report

#### ⚖️ Legal
- Проверяет лицензии зависимостей
- Оценивает юридические риски
- Готовит контракты и соглашения
- Результат: compliance report

#### 📣 Marketing
- Marketing Lead разрабатывает стратегию продвижения
- Content Manager готовит материалы (статьи, кейсы)
- Growth Manager запускает кампании (если нужно)
- Brand Designer готовит визуальные материалы
- Результат: GTM strategy, маркетинговые материалы

### Шаг 3: Сборка результата (CEO)
- Собрать артефакты от всех отделов
- Проверить целостность
- Сформировать итоговый отчёт
